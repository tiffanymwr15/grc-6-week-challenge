# Week 2: Make the Rules Executable

Three Rego policies in Open Policy Agent (OPA) read a Terraform plan JSON and deny non-compliant changes before apply. **SC-28** denies any `aws_s3_bucket` without a matching `aws_s3_bucket_server_side_encryption_configuration`, matched by resource reference (not bucket name, which is unknown at plan time). **AC-3** denies buckets without a complete `aws_s3_bucket_public_access_block` (all four flags true in `planned_values`). **CM-6** denies taggable resources (e.g. `aws_s3_bucket`) missing any of `Project`, `Environment`, `ManagedBy`, or `ComplianceScope` in `tags_all`. Six unit tests (two per control) define done; `opa test policies/ -v` shows `PASS: 6/6`. Conftest gates the real Week 1 plan. Most GRC portfolios describe controls in prose — this one expresses them as testable code that runs in milliseconds.
