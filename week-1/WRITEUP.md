# Week 1: Your First Compliant Resource

This Terraform configuration enforces five NIST SP 800-53 Rev. 5 controls on a pair of AWS S3 buckets: a primary data bucket and a dedicated access-log bucket. **SC-28 (Protection of Information at Rest)** applies AES-256 server-side encryption to both buckets by default. **AC-3 (Access Enforcement)** blocks public access on all four AWS vectors on both buckets (`block_public_acls`, `block_public_policy`, `ignore_public_acls`, and `restrict_public_buckets` are each hardcoded to `true`). **CM-6 (Configuration Settings)** enables versioning on the primary bucket so prior object states remain recoverable and auditable, and applies four required tags (`Project`, `Environment`, `ManagedBy`, `ComplianceScope`) to every taggable resource through the provider `default_tags` block. **AU-3 (Content of Audit Records)** and **AU-6 (Audit Record Review)** route primary-bucket access logs to the log bucket using ownership controls, a `log-delivery-write` ACL, and bucket logging. Compliance proof is machine-readable: `terraform show -json` produces `evidence/plan.json` with the encryption rule, public access block, tags, versioning, and logging target, and the `encryption_algorithm` output attests SC-28 in plain text (`AES256`).
